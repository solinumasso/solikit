📖 Documentation

Comment fonctionne le matching entre la base FINESS (établissements sanitaires et médico-sociaux français) et l'API Soliguide. Toutes les règles, dictionnaires et algorithmes utilisés pour produire les données affichées.

🎯 Vue d'ensemble

L'objectif est de rapprocher chaque établissement FINESS d'une fiche Soliguide existante quand c'est possible. Cela permet à Soliguide de tenir à jour son référentiel à partir de la source d'autorité officielle, et de repérer les structures qui mériteraient une fiche mais n'en ont pas encore.

Le pipeline en 4 étapes :

  1. Téléchargement du CSV FINESS (mise à jour quotidienne ATLASANTÉ) + des fiches santé Soliguide via l'API.
  2. Normalisation + géocodage des adresses via la Géoplateforme (BAN).
  3. Matching ville par ville selon un score composite (nom, adresse, GPS, code postal, téléphone).
  4. Enrichissement via DILA (Annuaire Service-Public) sur les structures à SIRET commun.

📚 Sources de données

  • FINESS (data.gouv.fr) — extract quotidien ATLASANTÉ, ~100 000 établissements.
  • Soliguide API — fiches santé en France (~6 500), API officielle authentifiée (JWT).
  • BAN / Géoplateforme (IGN) — géocodage et normalisation d'adresses, 40 appels/s.
  • DILA (Annuaire Service-Public) — 94 000 structures administratives, jointure par SIRET.

🗂️ Catégories FINESS retenues

Sur les ~100 000 établissements FINESS, on filtre pour garder uniquement les 34 catégories pertinentes pour un référentiel solidaire (santé, social, médico-social), ce qui réduit à ~42 000 fiches comparables.

⚙️ Algorithme de matching

Pour chaque FINESS retenu, on cherche le meilleur candidat Soliguide dans la même ville. Un score composite est calculé entre 0 et 1, à partir de 5 signaux :

SignalPoids (avec GPS)Poids (sans GPS)
Similarité du nom30 %35 %
Similarité d'adresse25 %35 %
Distance GPS (≤200 m)25 %—
Code postal identique10 %15 %
Bonus téléphone identique+10 %+15 %

Confiance attribuée

  • Certain — score ≥ 75 %
  • Possible — score 60 à 75 %
  • Non matché — score < 60 %, ou si nomLev < 70 % ET adresseLev < 70 % sans téléphone

La similarité du nom et de l'adresse utilise le max entre Levenshtein (par caractères) et un containment par tokens (fraction de tokens partagés). Le containment tolère les info supplémentaires d'un côté.

🏷️ Normalisation des noms

Avant comparaison, chaque nom passe par une pipeline :

  1. Minuscules, suppression des accents, des ponctuations et guillemets
  2. Dédoublonnage acronyme/expansion : si "LHSS Lits Halte Soins Santé" est là, on garde une seule des deux
  3. Expansion d'acronymes (~40 connus, voir ci-dessous)
  4. Synonymes multi-mots (ex: "centre hospitalier" → "hopital")
  5. Retrait des mots vides : d, l, de, du, des, le, la, les, a, au, aux, et

📍 Normalisation des adresses

Les adresses passent par :

  1. Géocodage Géoplateforme (BAN) : transforme une adresse brute en label normalisé + INSEE + GPS précis
  2. Expansion des types de voie abrégés (~30 mappings, conventions BAN)
  3. Retrait des mots vides

🇫🇷 Enrichissement DILA

Pour chaque FINESS avec un SIRET, on tente une jointure exacte sur le SIRET avec l'Annuaire Service-Public (DILA). Quand ça matche, on récupère :

  • 🌐 Site(s) web
  • ✉️ Email
  • 📞 Téléphone(s) au format E.164 (+33...)
  • 🕒 Horaires structurés (objet jour-par-jour) — uniquement pour les fiches saisies en direct par l'administration, pas pour les fiches reprises de data·inclusion
  • 🔗 URL canonique sur lannuaire.service-public.gouv.fr
  • 🏷️ Provenance (partenaire) : data·inclusion, Ma Boussole Aidants, ministère de la Justice…

L'onglet 📋 DILA du dashboard montre toutes les fiches enrichies. Un badge 📋 apparaît à côté du nom dans les autres onglets.

🎛️ Les onglets du dashboard

  • 🔗 Matchs — toutes les correspondances trouvées (certains + possibles), triées par score
  • 🔄 À mettre à jour — sous-ensemble des matchs certains où la fiche FINESS a une date de mise à jour plus récente que celle de Soliguide : potentiels candidats à la propagation
  • 📋 DILA — les matchs enrichis via SIRET avec données DILA. Filtre par fournisseur disponible
  • 🏥 FINESS non matchées — fiches FINESS sans correspondance Soliguide. Cliquer "+" affiche un appel API live pour les 5 Soliguide les plus proches géographiquement
  • 📍 Soliguide non matchées — fiches Soliguide sans correspondance FINESS (peut-être un FINESS qui n'a pas été retenu par la whitelist, ou une fiche associative pure)

Le filtre par département en haut applique à tous les onglets. Le filtre confiance n'agit que sur l'onglet Matchs.
