# ABS-CCAS Implementation Plan

**Goal:** Créer un outil interactif permettant aux CCAS de comparer des indicateurs Soliguide entre leur commune et d'autres territoires (EPCI, département, région, autres communes) pour réaliser leur Analyse des Besoins Sociaux.

**Architecture:** L'outil est composé de deux pages statiques. La page 1 (`index.html`) permet de sélectionner les territoires à comparer ; la sélection est passée via URL params à la page 2 (`results.html`) qui affiche un tableau comparatif avec des graphiques d'évolution temporelle dépliables. Un script TypeScript génère des données mockées en JSON, que le dashboard lit via `fetch()`.

**Tech Stack:** TypeScript + tsx (script mock), HTML/CSS/JS vanilla, DaisyUI 5 + Tailwind 4 CDN, Chart.js CDN, thème Soliguide partagé.

## Structure du JSON mocké

Le script génère `mock-data.json` avec cette forme :

```json
{
  "communes": [
    {
      "code": "75056",
      "nom": "Paris",
      "epci": { "code": "200054781", "nom": "Métropole du Grand Paris" },
      "departement": { "code": "75", "nom": "Paris" },
      "region": { "code": "11", "nom": "Île-de-France" }
    }
  ],
  "territoires": {
    "75056": {
      "code": "75056",
      "nom": "Paris",
      "type": "commune",
      "indicateurs": {
        "nb_services_alimentaires": {
          "valeur": 42,
          "historique": [{ "mois": "2022-01", "valeur": 38 }, ...]
        },
        "taux_ouverture": {
          "valeur": 78.5,
          "historique": [{ "mois": "2022-01", "valeur": 72.0 }, ...]
        },
        "taux_saturation": {
          "valeur": 65.2,
          "historique": [{ "mois": "2022-01", "valeur": 58.0 }, ...]
        },
        "top_categories": {
          "valeur": ["Alimentation", "Hébergement", "Santé"],
          "historique": null
        }
      }
    },
    "200054781": { ... },
    "75": { ... },
    "11": { ... }
  }
}
```

Les `historique` couvrent 48 mois (janvier 2022 → décembre 2025). `top_categories.historique` est `null` (pas de série temporelle pour cet indicateur).

## Navigation entre les pages

Page 1 → Page 2 via URL params :
```
results.html?territories=75056,200054781,75,11
```
`results.js` parse `URLSearchParams`, charge `mock-data.json`, et construit le tableau dynamiquement.

  **Génération des indicateurs** : pour chaque territoire (commune + EPCI + département + région), générer avec `Math.random()` des valeurs réalistes et une série temporelle de 48 mois (2022-01 → 2025-12) :

  - `nb_services_alimentaires` : entier entre 1 et 80 (communes), 10-200 (EPCI), 20-400 (dept), 50-1000 (région)
    - historique : variation aléatoire ±10% par mois, tendance légèrement croissante
  - `taux_ouverture` : float 40.0-95.0 (%)
    - historique : variation ±5 points, saisonnalité (légère baisse en été)
  - `taux_saturation` : float 30.0-90.0 (%)
    - historique : variation ±7 points, saisonnalité (hausse en hiver)
  - `top_categories` : tirage parmi `["Alimentation", "Hébergement", "Santé", "Emploi", "Administratif", "Petite enfance", "Hygiène", "Transport"]`, 3 valeurs sans doublon
    - `historique: null`

  **Seed déterministe** : utiliser un seed basé sur le code du territoire pour que les données soient stables entre les regénérations (évite les diffs inutiles). Exemple avec une simple fonction LCG :
  ```ts
  function seededRandom(seed: number) {
    let s = seed;
    return () => {
      s = (s * 1664525 + 1013904223) & 0xffffffff;
      return (s >>> 0) / 0xffffffff;
    };
  }
  ```
