Règles & cadre

Les 7 règles obligatoires du projet. À respecter pour chaque outil.

📦

Scripts → JSON → Dashboard

Les scripts TypeScript font tout le travail lourd (téléchargement, parsing, calculs) et produisent du JSON dans data/output/. Le dashboard ne fait que lire et afficher ce JSON. Jamais de CSV ou de calculs côté navigateur.

🎨

Thème Soliguide obligatoire

Chaque dashboard utilise assets/soliguide-theme.css avec data-theme="soliguide". Ne jamais redéfinir les couleurs ou le thème. Utiliser les classes DaisyUI, pas des hex hardcodés.

🚀

HTML/CSS/JS pur, pas de framework

Pas de React, Vue, Svelte ou Angular. Les dashboards sont en vanilla JS avec DaisyUI 5 + Tailwind CSS 4 via CDN. Déployables directement sur GitHub Pages, sans build step.

🗂️

Données publiques, jamais committées

Les données viennent de sources ouvertes (data.gouv.fr, API publiques). Le dossier data/ est dans .gitignore. Chaque projet doit permettre de re-télécharger les données depuis les scripts.

🔤

Polices Roboto & Nunito uniquement

Roboto pour le corps de texte et les données. Nunito pour les titres. Via Google Fonts CDN. Aucune autre police autorisée.

🖼️

Logo & favicon Soliguide

Chaque dashboard affiche le logo Soliguide dans la navbar (en couleurs naturelles) et utilise le favicon. Les assets sont dans assets/images/ et copiés localement dans dashboard/assets/.

⌨️

pnpm dashboard sur le port 5555

Chaque package.json a un script "dashboard": "npx -y serve ../.. -p 5555". Le serveur sert depuis la racine du repo pour que les chemins ../../assets/ fonctionnent.

En résumé

Faire

  • Utiliser les classes DaisyUI pour les couleurs
  • Produire du JSON en sortie des scripts
  • Numéroter les scripts (01-, 02-, 03-…)
  • Copier les assets Soliguide dans dashboard/assets/
  • Ajouter le lien doc dans le menu hamburger

Ne pas faire

  • Hardcoder des couleurs hex dans le CSS
  • Utiliser un framework JS (React, Vue…)
  • Parser du CSV côté navigateur
  • Committer des fichiers de données
  • Utiliser des polices autres que Roboto/Nunito
