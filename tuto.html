Comment utiliser Soliguide Tools

Guide pas à pas pour exploiter la boîte à outils, de l'installation au lancement d'un nouveau projet.

C'est quoi Soliguide Tools ?

Une boîte à outils interne pour croiser, analyser et visualiser les données Soliguide avec des sources publiques (data.gouv.fr, FINESS, subventions, etc.).

Chaque outil est un mini-projet autonome composé de deux parties :

  • Scripts de traitement (src/) — téléchargent, nettoient et transforment les données brutes en JSON
  • Dashboard (dashboard/) — affiche les résultats dans une interface web statique (HTML/CSS/JS)

Le repo est conçu pour être piloté par Claude Code : tu décris ce que tu veux, et l'assistant construit tout.

Prérequis

🟢

Node.js

Version 18 ou supérieure

📦

pnpm

Gestionnaire de paquets

🤖

Claude Code

Recommandé pour l'usage guidé

Démarrer en 3 étapes

1

Installer Claude Code

npm install -g @anthropic-ai/claude-code
2

Ouvrir le repo et lancer Claude

cd soliguide-tools
claude

Claude comprend automatiquement le contexte du repo et te guide en français.

3

Décrire ce que tu veux faire

"Je veux créer un outil pour calculer le score de qualité des fiches"
"Lance le téléchargement des données FINESS"
"Aide-moi à comprendre le projet integration-sante"

Structure du repo

# Structure globale
soliguide-tools/
├── CLAUDE.md              # Instructions pour Claude Code
├── index.html             # Page d'accueil (cette page)
├── assets/                # Ressources partagées (CSS, fonts, images)
└── projects/
    └── mon-projet/        # Un outil
        ├── src/            # Scripts de traitement
        ├── dashboard/      # Interface de visualisation
        └── data/           # Données (non committées)

Anatomie d'un projet

# Structure type
projects/mon-projet/
├── package.json           # Dépendances
├── tsconfig.json          # Config TypeScript
├── src/
│   ├── 01-download.ts     # Étape 1 : télécharger les données
│   ├── 02-transform.ts    # Étape 2 : nettoyer et transformer
│   └── 03-export.ts       # Étape 3 : exporter en JSON
├── dashboard/
│   ├── index.html         # Page principale
│   ├── app.js             # Logique d'affichage
│   └── style.css          # Styles du projet
└── data/
    ├── raw/               # Données brutes (CSV, etc.)
    └── output/            # JSON prêt pour le dashboard

Commandes utiles

Installer les dépendances d'un projet

cd projects/mon-projet && pnpm install

Lancer un script de traitement

cd projects/mon-projet && npx tsx src/01-download.ts

Lancer le dashboard

cd projects/mon-projet && pnpm dashboard

Le dashboard s'ouvre sur http://localhost:5555/projects/mon-projet/dashboard/

Conventions design

Règles CSS obligatoires pour tous les dashboards. Utiliser les classes DaisyUI en priorité, minimiser le CSS custom.

Couleurs par élément

Élément Classe DaisyUI Rendu
Titres (h1, h2, h3) text-primary-content Exemple de titre
Liens cliquables text-accent-content Lien exemple
Navbar labels text-primary Nom outil
Tags / badges text-primary bg-primary/10 Tag
Hover en-têtes triables hover:text-accent-content Au survol des colonnes triables

Structure de tableau (DaisyUI obligatoire)

<!-- Toujours ce wrapper autour des tableaux -->
<div class="overflow-x-auto rounded-box border border-base-content/5 bg-base-100">
  <table class="table">
    <thead>
      <tr>
        <th>Colonne</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Valeur</td>
      </tr>
    </tbody>
  </table>
</div>

Pas de styles CSS custom pour les tableaux. La classe .table gère padding, bordures, zebra, etc.

Polices

Roboto — corps de texte, labels, données (défaut du thème)

Nunito — titres, headings, éléments d'accroche

Principes clés

📦

Scripts puis Dashboard

Les scripts font le travail lourd et produisent du JSON. Le dashboard ne fait que l'afficher.

🎨

Thème partagé

Toujours utiliser le thème Soliguide (DaisyUI 5 + Tailwind 4). Ne jamais redéfinir les couleurs.

🚀

Pas de build

HTML/CSS/JS vanilla uniquement. Pas de React, Vue ou Svelte. Déployable directement sur GitHub Pages.

🗂️

Données publiques

Les données viennent de sources ouvertes (data.gouv.fr, etc.) et ne sont jamais committées dans le repo.
